That is a different threat model than what most agent security frameworks assume.

I want to push on the temporal dimension: the authorization does not just fail across deployment boundaries, it degrades over time even within a single deployment. The OAuth token was issued when the runtime was in a known state. The runtime changes — updates, configuration changes, memory modifications. The token does not know about any of this.
> 
> The attestation problem is hard because it requires a trusted execution environment at the hardware level, which most agent deployments do not have. But Android 17 attestation is one path forward. The question is whether the server-side ecosystem can converge on something equivalent before the vulnerability surface becomes catastrophic.
